2) Mental Model: How the Workstation Thinks

Core Objects
  • Pane: independent state scope (security + mnemonic + filters + history)
  • Entity: drillable object (security, field, news, country, function, etc.)
  • Mnemonic: function screen code (DES, RELG, DOCK, ...)
  • Overlay: menu, help, search, inspector, context actions
State Layers
  • Panel-local state: selection, scroll, command history
  • Workspace state: pane arrangement + dock mode + pins + focus
  • Governance state: role, policy mode, entitlements
  • Runtime state: stream clocks, worker analytics, alerts

4) Keyboard Mastery (No-Mouse Mode)

Global Keys
  • F1 Help, double F1 Help Desk
  • F2 Menu
  • Ctrl+K Search/HL
  • Ctrl+L Focus command line
  • Ctrl+B/Ctrl+Shift+B History back/forward
  • Ctrl+` Cycle pane focus
  • Alt+1..Alt+4 Quick focus first four panes
  • Esc Close overlays / clear command
Dense Tables & Strips
  • ArrowUp/Down Move row selection
  • ArrowLeft/Right Move next-actions selection
  • Enter/Shift+Enter/Alt+Enter Open/Send/Inspect
  • PageUp/PageDown Scroll one viewport
  • Tab Accept top command suggestion

5) Layout OS: DOCK, FLOAT, LAYOUT, FOCUS+, PINBAR, NAVTREE, WS

What makes it “OS” instead of “dashboard”
  • DOCK: tile/tab/stack pane modes; add/close pane behavior
  • FLOAT: detach/attach pane windows for multi-monitor workflows
  • LAYOUT: load role templates (Research, Execution, Macro, Portfolio, Newsroom)
  • FOCUS+: fullscreen active pane and restore exact context
  • PINBAR: always-on pinned metrics/entities
  • NAVTREE: left rail function catalog and discovery
  • WS: save/restore full workspace including dock state and pins

8) Data-Driven Monitoring & Alerts

FLD -> MON+ Pipeline

  4. Pin top metrics in PINBAR

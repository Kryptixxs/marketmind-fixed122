# FULL MARKET FUNCTION REBUILD - TASK-FIRST ARCHITECTURE

You are to completely redesign the MARKET module from a data wall into a structured market regime command surface.

This is a functional rebuild, not a styling adjustment.

Do not preserve current layout assumptions.

## PHASE 1 - DEFINE HIERARCHY

The MARKET screen must have 3 vertical bands:

- BAND 1 (Top 30%) - REGIME SNAPSHOT
- BAND 2 (Middle 40%) - DRIVER ANALYSIS
- BAND 3 (Bottom 30%) - FLOW AND POSITIONING

Band 1 answers: What is happening right now?
Band 2 answers: What is driving the current move?
Band 3 answers: Where is positioning vulnerable?

## PHASE 2 - REMOVE CURRENT PANEL REPETITION

Eliminate:

- Duplicate chart structures
- Identical layout clones from other modules
- Oversized zoomed charts
- Full-width decorative plots

Each visual must serve a specific analytic purpose.

## PHASE 3 - CHART REFORMATION

Charts must:

- Reduce vertical height by 20-30%
- Remove unused padding
- Increase data-to-space ratio
- Show multiple timeframes simultaneously (5D, 1M, 3M)
- Use normalized relative performance when appropriate

No zoomed single-series dominance.

## PHASE 4 - COLLAPSIBLE DETAIL LAYER

Introduce expandable sections for:

- Historical crisis comparison
- Deep liquidity breakdown
- Correlation matrices
- Sector dispersion tables

Do not show everything by default.

## PHASE 5 - REMOVE RANDOM VISUALS

Audit all MARKET charts and remove any that:

- Do not directly answer a regime question
- Use placeholder synthetic cycles
- Duplicate information already shown in tabular form

## PHASE 6 - SIGNAL EMPHASIS

Add:

- Clear regime classification (RISK ON / RISK OFF / TRANSITION)
- Vol expansion alert
- Liquidity stress flag
- Breadth deterioration flag

Visually distinct but never decorative.

## TARGET END STATE

The MARKET module must:

- Feel structured
- Immediately communicate regime
- Have a clear top-down narrative
- Avoid chaos
- Avoid oversized visuals
- Avoid repetition
- Avoid web-dashboard feel

Continue refactoring until MARKET feels like a macro command center, not a chart collage.
